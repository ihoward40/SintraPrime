# Default Email Sub-Protocol (v1.1)

**Purpose**
Convert reader clarity into a single economic action.

## Mandatory Structure

1. **Explicit Problem Declaration**
   Name the pain in plain language. No metaphors.

2. **Operational Reframe**
   One insight that changes how the reader understands the problem.

3. **Single Next Step**
   One action only (reply, click, book).

## Prohibited

- Multiple CTAs
- Teaching without direction
- Topic blending
